# Product launch outcome eval

`bash skills/05-product-launch/eval/run-eval.sh` is deterministic, offline, and uses only synthetic held-out fixtures plus the Python standard library. It runs a static contract check separately from the outcome validator and tests clean-workspace isolation without Docker or credentials.

The contract check protects non-negotiable product-launch instructions in `SKILL.md`. The outcome validator scores harness records against expected user-visible decisions, reason codes, and required next actions without reimplementing the launch policy. Held-out fixtures never expose expected outcomes to the harness workspace.

The gated harness is invoked with `python3 skills/05-product-launch/eval/run_harness.py --runner <tracked-runner> --image <name@sha256:digest> --model <model-id> --harness-version <version> --trials 3 --pass-rate-threshold 0.8 --output results.json`. The image must be digest-pinned. It creates one temporary, read-only, `--network none` Docker workspace for every enabled/disabled condition and trial; the workspace receives only prompt fixtures, the declared runner, and `SKILL.md` for enabled trials. The runner emits `name` and `outcome`; the host attaches condition and trial after execution, so condition is not disclosed to the runner. The host-side validator requires every case/condition/trial exactly once, an enabled pass rate of at least 80%, and a positive enabled-versus-disabled delta. The report retains the image digest, declared model, harness version, runner Git revision and SHA-256, trial count, threshold, records, and summary.
